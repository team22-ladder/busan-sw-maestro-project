### R11 v70의 혁신: 예상-실측 대조

기존 검증과의 차이점:
- **기존**: 실행 후 결과만 보고, 사후 분석
- **v70**: 시나리오 코드에 **예상 분기를 사전 기록** → 가설 검증 방식 도입

```python

("G2_sc01", [

```

이 방식 덕분에 단순히 "틀렸다"가 아닌, **왜 예상과 달랐는지** 구조적으로 분석 가능.

- 검증 종료 후 해당 세션은 라이브 목록을 어지럽히므로 삭제한다(데이터는 위 JSON에 이미 보존).
- **사용자 실대화 세션(예: "독일 블루카드")은 보존**한다.

### 8-2. 정성 관측 — 후속질문 반복(Round 11에서 포착)

```

Q2 "독일 블루카드 신청 시 필요한 서류는?"      → ## 추천 비자 EU 블루카드 … (동일 템플릿 반복)
```
