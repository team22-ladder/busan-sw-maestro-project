## Round 11.1 — 후속질문 응답 압축 워크플로우

### 문제
같은 비자(국가+목적)에 대한 후속·상세 질문(필요서류·대체조건·처리기간 등)에도 매번
`## 추천 비자 / 주요 자격 요건 / 필요 서류 …` **전체 템플릿을 반복 출력**해 정보가 중복됨.
(예: "독일 블루카드 자격요건?" → "필요 서류?" → "B1 미만 대체조건?" 모두 동일 템플릿)

### 해결 — `is_followup` 기반 압축 응답
- **감지(`intent.py`)**: 이전 대화에서 이미 비자가 정해진 뒤의 추가/상세 질문을
  `is_followup=true`로 분류(신규 추천 요청은 false).
- **압축(`agent/nodes/response.py`)**: `is_followup`이면 전체 템플릿을 생략하고
  **물어본 부분만** 간결히 답하며, **상위 대화를 참조**("앞서 안내드린 …비자 기준으로").
  실패 시 표준 템플릿으로 폴백.
- **표시(`web/public/app.html`)**: 후속 답변은 **라이트 카드(`doc-light`)** — 직인·분류표·
  표지 생략, 본문만 표시.
- **트레이스**: response_formatter 노드 디테일에 "후속질문 압축 응답(상위 대화 참조)" 표기.

### 검증 (`tests/run_compression_verify.py`)
독일 블루카드 체인 + **국가 전환 반대사례**로 6턴 검증 → **6/6 통과**:

| 턴 | 질문 | is_followup | 결과 |
|----|------|:-----------:|------|
| T1 | 독일 블루카드 자격 요건? | False | 전체 템플릿(1760자) |
| T2 | 필요 서류는? | True | 압축(547자)·상위참조 |
| T3 | 독일어 B1 미만 대체조건? | True | 압축(350자)·상위참조 |
| T4 | 처리 기간은? | True | 압축(229자)·상위참조 |
| **T5** | **그럼 캐나다 취업은?** | **False** | **전체 템플릿(1695자)** — 새 주제는 압축 안 함 |
| T6 | 캐나다 그 비자 서류는? | True | 압축(509자)·상위참조 |

→ **평균 76.8% 길이 감소**, 새 주제(T5)는 정상 추천 유지(무조건 압축 아님).

## 추가 개선(5차) — AI 후속칩·반응형 트레이스 레이아웃·종합 검증

- **트레이스 그래프 반응형 배치 + 사용자 줌**: 공간이 충분하면 가로(LR), 좁으면 세로(TB)로 자동

## 추가 개선(4차) — 세션 영속·후속요청·세션별/통합 트레이스

- **선택 노드/간선 강조**: 실행 경로 노드는 강한 글로우 + 다크 헤일로 + 약간 확대로 최상위
  z-index 렌더.
- **후속 요청 제안(ChatGPT 식)**: 답변 직후 `더 구체적으로 / 최신 정보 / 다른 비자 종류 / 공식
  사이트 상세 탐색` 버튼. 클릭 시 사용자가 입력한 것처럼 자동 실행(멀티턴 맥락 활용).
- **공식 사이트 상세 탐색(딥서치)**: '상세 탐색/공식 사이트/최신 정보' 요청 시 RAG 를 건너뛰고
  `web_search_tool` 직행 + Tavily `search_depth=advanced` + 원문(raw_content)으로 깊게 탐색.
- **대화 영속화(새로고침 보존)**: 백엔드 RESTful 세션 스토어(`/sessions`, `data/sessions.json`).
  활성 세션은 URL `?sid=` 로 유지.
- **세션 메타데이터**: 이름 변경·한줄 설명·태그(장기체류/취업/유학/여행…) 편집·표시.
- **세션별 RESTful 트레이스(2a)** + **통합 병렬 허브(2b)**: 세션은 `?sid=<id>` ↔ `/{id}/trace`
  1:1 매핑. `/trace` 는 모든 세션을 카드로 한눈에.

- **의도 분류 강건화**: `intent_classifier`를 결정적(temperature=0)으로 실행, 도메인 키워드·
  국가/목적 신호가 있으면 LLM 오판을 무시하고 비자 플로로 강제.
- **멀티 채팅 세션**: 사이드바 ➕ 새 대화 생성·전환·삭제, 대화별 맥락 유지.
- **Tavily 검색어 정제**: 한국어·`Korea` 배제, 핵심 영어 쿼리(국가명 + visa + requirements/
  eligibility/official)만 사용.

## 데모 리뷰 반영(개선 이력)

- **장기 체류 비중 강화**: 단기→영주권·정착 경로와 갱신/전환 조건 함께 안내.
- **교차 예외규칙 보강**: 쉥겐↔비쉥겐, 환승, ESTA/eTA, 유효기간≠체류 등 13종 규칙을 RAG 적재.
- **전세계 국가 대응**: 6개국은 RAG, 그 외는 `search_hints` 의 우선 공식도메인·검색어 템플릿.
- **검색 신뢰도 게이트 + 검색어 재생성 루프**: 공식 출처 부족 시 LLM이 영어 검색어 재생성.
- **일반 대화 분기**: 비자 무관 질문을 `general_chat` 로 분리.
- **멀티턴 맥락 / 응답 토큰 스트리밍 / SPA ↔ /trace 실시간 연동**.
