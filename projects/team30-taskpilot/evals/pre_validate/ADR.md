# ADR-001: pre_validate 실제 LLM 성능 평가 방식

## Status
Accepted

## Context
단위 테스트는 LLM을 mock하므로 실제 모델이 일정 유효성, 시간 범위, 일정 충돌을 올바르게 판단하는지 측정할 수 없습니다. 실제 LLM 출력은 실행마다 달라질 수 있으므로 한 번의 통과만으로 안정성을 판단하기 어렵습니다.

## Decision
- `pre_validate_schedule`을 직접 호출하는 실제 LLM 평가를 그래프 테스트와 분리한다.
- 합성·익명화된 JSONL 케이스로 유효성 판단, 실패 사유, 정규화 필드, 위치 간 이동 가능성, 장소 제약 확인 질문을 평가한다.
- 평가 실행에서는 `strict=True`를 사용해 모델 호출 및 구조화 출력 오류를 별도 실패로 집계한다.
- 기본 5회 반복 실행하며, 모든 케이스가 모든 반복에서 통과해야 안정적이라고 판단한다.
- 평가 코드와 케이스는 커밋하고 `results` 디렉터리의 실행 결과는 커밋하지 않는다.
- 초기에는 점수를 리포트만 하고 CI 게이트로 사용하지 않는다.

## Consequences
- 노드 담당자가 다른 노드와 독립적으로 실제 모델 성능을 확인할 수 있다.
- 팀원이 동일한 케이스와 점수 계산 방식으로 결과를 재현할 수 있다.
- 실제 LLM 평가에는 API 키, 비용, 네트워크 연결이 필요하다.
- 반복 평가로 인해 단일 실행보다 비용과 시간이 증가한다.
