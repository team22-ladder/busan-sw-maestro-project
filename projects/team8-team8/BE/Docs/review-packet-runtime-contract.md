## Runtime Boundary Audit

- Production dialogue path uses configured `AIClient` against `/internal/v1/dialogue/respond`.
- Missing AI base URL, timeout, connect error, non-2xx, invalid shape, or AI degraded metadata maps to explicit degraded result.
- Dialogue degraded result raises 503 `AI_SERVICE_DEGRADED` before dialogue log save, question consumption, notes, unlocks, tension, or events.
- AI payload, AI answer/proposedEvents, applied events, merged public session payload, and runtime SSE smoke are checked for canonical forbidden ref tokens.
- AI answer/proposedEvents with forbidden refs are rejected as 503 `AI_RESPONSE_FORBIDDEN_REF` before session persistence or SSE.
- User-submitted final accusation motive/method is checked before `RuleEngine` mutates session state; forbidden refs return 400 and do not persist accusation state or emit SSE.
- Test fakes are isolated in `tests/test_api_smoke.py`; they are dependency overrides only.
- Debug endpoints are gated by `BE_DEBUG_TOOLS_ENABLED`; docker compose enables them for local dev/runtime dogfood only.
- No silent local/canned dialogue progress is accepted when provider fails.

## Event Processor Audit

- Dialogue EventProcessor now disables implicit note creation from BE fallback text.
- With `allowed_event_types` present, AI proposedEvents must be in `GAME_MASTER_PUBLIC_EVENT_TYPES`:
  - `NOTE_FACT_ADDED`
  - `NOTE_CONTRADICTION_CANDIDATE_ADDED`
  - `BOOKMARK_SUGGESTED`
- Proposed note events must match this turn's `allowedEventPolicy` related IDs, not merely any visible session ID.
- `TENSION_CHANGED` from AI is rejected.
- Visual state is generated by BE, not accepted as AI-owned state mutation in dialogue policy.
- Evidence/timeline unlock SSE is derived from BE rule/session unlock state, not AI authority.
- Contradiction candidate events require visible required statement/evidence IDs and include canonical `contradictionId`, `statementIds`, `evidenceIds`, `timelineIds`, `submitEligible`.

## Public Leak Audit

Fixed public stable IDs:
- `st_choiyuna_secret_call` -> `st_choiyuna_call_record`
- `con_secret_call` -> `con_call_record`

Public safety payload now uses neutral fields and does not expose internal `leaksSolution`.

Public final accusation result no longer exposes `culpritCorrect`; it uses neutral `suspectMatch` internally/publicly and emits `ACCUSATION_RESOLVED` SSE only after the public event payload passes the guard.
