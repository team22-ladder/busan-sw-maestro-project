# Senario Docs

이 폴더는 Scene별 범인을 추리하는 앱의 사건 콘텐츠를 정의한다. `Senario` 표기는 요청 폴더명을 그대로 따른다.

## 문서 구성

| 문서 | 역할 |
| --- | --- |
| `scene-template.md` | Scene 하나를 만들 때 채워야 하는 전체 템플릿 |
| `case-001.md` | MVP 샘플 사건 정의 |
| `schema.md` | 사건 데이터에 필요한 공통 ID와 필드 규칙 |

## Scene 정의에 반드시 필요한 항목

| 항목 | 설명 |
| --- | --- |
| Scenario | 사건 배경, 피해자, 발생 장소, 발생 시각, 목표 |
| Characters | 용의자와 주요 인물의 이름, 역할, 성격, 숨기는 정보 |
| Relationships | 인물과 피해자 사이의 관계, 갈등, 동기 후보 |
| Evidence | 증거, 기록, 물증, 목격담, 발견 조건 |
| Character Timelines | 인물별 시간대 이동과 진술 |
| Contradictions | 진술과 증거 또는 진술과 진술 사이의 모순 |
| Solution | 실제 범인, 동기, 수단, 핵심 근거 |

## 작성 원칙

- 모든 인물, 증거, 진술, 모순은 고유 ID를 가진다.
- 플레이어에게 처음부터 공개되는 정보와 해금 후 공개되는 정보를 분리한다.
- 정답은 범인 이름 하나가 아니라 `범인 + 동기 + 수단 + 핵심 모순` 조합으로 정의한다.
- 타임라인은 최소 사건 전 30분부터 사건 후 30분까지 작성한다.
- 모순은 BE Rule Engine이 판정할 수 있도록 관련 진술 ID와 증거 ID를 연결한다.
