## 4. 기술 메모

- **MySQL 드라이버**: 동기 SQLAlchemy + `pymysql`(MVP). 저장은 짧은 연산.
- **스키마**: Alembic이 단일 소스. 팀원은 `alembic upgrade head`. (create_all 미사용)
- **Qdrant 차원**: 임베딩 샘플로 측정해 생성(하드코딩 금지).
- **적재**: 50개 `--limit` 시범 적재로 속도 보정 → 전체 풀런 → 공유 Qdrant Cloud. 멱등 upsert라 재실행 안전.
- **best-effort 저장**: DB 실패해도 응답은 막지 않음.
- **LLM**: Upstage Solar(`solar-pro2`) 유지.

## 5. 검증 (end-to-end)

1. `cp .env.example .env` → `UPSTAGE_API_KEY`, `QDRANT_URL/API_KEY`, `WIKI_VAULT_PATH` 입력
2. `docker compose up -d mysql qdrant` → `docker compose ps` healthy
3. `uv sync && uv run alembic upgrade head` → MySQL에 3테이블 생성 확인
4. `uv run python scripts/ingest_wiki.py --limit 50` → 속도·정상 동작 보정(소요시간 환산)
5. `uv run python scripts/ingest_wiki.py` → 전체 적재(공유 Qdrant) — 콜렉션 count 확인
6. `uv run uvicorn app.main:app --port 8001` → `GET /health` → `healthy`
7. `curl -X POST localhost:8001/api/v1/chat/sync -d '{"message":"방금 마크 시작했는데 뭐부터 해야 해?"}' -H 'Content-Type: application/json'` → 위키 근거 기반 초보 코치 응답
8. `docker compose exec mysql mysql -uapp -papp minecraft` → `SELECT * FROM chat_sessions; SELECT * FROM messages;` 행 확인
9. `uv run streamlit run frontend/ui.py --server.port 8002` → 브라우저 동작(PR 스크린샷)
10. (협업 검증) 새 클론에서 `cp .env.example .env && alembic upgrade head` → 동일 스키마 재현 확인
