# 웹뷰 사용법 — Streamlit 검증용 챗봇

> 챗봇 동작을 **검증**하기 위한 Streamlit 웹 UI 사용법이다(`project_code/frontend/ui.py`).
> 실제 메인 클라이언트는 Fabric 모드이며([인게임_사용법.md](인게임_사용법.md)), 웹뷰와 모드는 동일한 백엔드 API를 공유한다. 기술 전반은 [기술_문서.md](기술_문서.md) 참고.

---

## 1. 실행

```bash
cd project_code
cp .env.example .env          # UPSTAGE_API_KEY + 공유 QDRANT_URL/API_KEY 입력
docker compose up -d mysql    # 로컬 MySQL
uv sync
uv run alembic upgrade head
bash start.sh                 # Backend → :8001 · 웹뷰 → http://localhost:8002
```

브라우저에서 **http://localhost:8002** 접속.
> 상세 절차·트러블슈팅은 [백엔드_사용법.md](백엔드_사용법.md) 참고.

## 2. 화면 구성

```
┌─────────────┬──────────────────────────────────────┐
│  사이드바    │  🧱 마인크래프트 초보 가이드          │
│             │                                      │
│ ➕ 새 대화   │   (대화 메시지 영역)                  │
│ ───────     │    🙂 사용자: ...                     │
│ 대화 목록    │    🤖 가이드: ...                     │
│  ▶ 현재대화  │       📚 참고한 위키 문서 ▼           │
│    이전대화🗑 │                                      │
│ ───────     │  [ 마인크래프트에 대해 물어보세요  ]   │
│ 세션: 1a2b… │                                      │
└─────────────┴──────────────────────────────────────┘
```

## 3. 대화하기

1. 하단 입력창에 질문을 입력하고 Enter.
   - 예: `방금 시작했는데 뭐부터 해야 해?`, `철 곡괭이 어떻게 만들어?`
2. 답변이 생성되는 동안 **진행 상태가 단계별로 표시**된다(SSE 스트리밍).

| 표시 | 내부 노드 | 의미 |
| --- | --- | --- |
| 🔍 질문 분석 중... | analyze | 도메인·키워드 분석 |
| 🤔 상황 파악 중... | clarify | 되묻기 필요 여부 판단 |
| 📚 위키 검색 중... | retrieve | Qdrant 위키 검색 |
| ✍️ 답변 생성 중... | respond | LLM 답변 생성(토큰이 실시간 출력) |
| 💬 되묻기 준비 중... | ask | 정보 부족 시 되묻기 |

3. 답변은 **토큰 단위로 실시간 출력**되며(`▌` 커서), 완료되면 커서가 사라진다.
4. 위키를 참고한 답변에는 하단에 **`📚 참고한 위키 문서`** 펼침 목록이 붙는다(검색에 사용된 문서 제목).

> 막연한 질문(예: "이제 뭐 해야 해?")에는 코치가 **되묻기**로 상황을 먼저 물을 수 있다. 이어서 상황을 답하면 단계별 안내로 이어진다.

## 4. 대화 목록 관리 (사이드바)

| 동작 | 방법 | 결과 |
| --- | --- | --- |
| **새 대화** | `➕ 새 대화` 버튼 | 새 `thread_id` 생성, 화면 비움 |
| **대화 전환** | 목록에서 대화 클릭 | 해당 세션의 전체 메시지를 불러와 표시 (`▶`가 현재 대화) |
| **대화 삭제** | 대화 옆 `🗑` 버튼 | 세션·메시지 삭제. 현재 대화를 지우면 새 대화로 전환 |

- 목록의 각 항목은 그 대화의 **첫 사용자 질문**을 미리보기로 보여준다(22자까지).
- 사이드바 하단에 현재 세션 ID 앞 8자리가 표시된다.
- 목록을 못 불러오면 "목록을 불러올 수 없습니다"가 뜬다(백엔드 `/api/v1/sessions` 확인).

> 대화는 백엔드 MySQL에 저장되므로, 웹뷰를 새로고침하거나 다시 켜도 사이드바 목록에서 이어볼 수 있다. (단, 새로고침 시 현재 화면의 메시지 영역은 비워지고 목록에서 다시 선택해야 한다.)

## 5. 웹뷰 vs 인게임 모드

| | 웹뷰(Streamlit) | Fabric 모드 |
| --- | --- | --- |
| 위치 | 브라우저 | 마인크래프트 인게임 |
| 용도 | **동작 검증용** | **실제 메인 클라이언트** |
| 호출 API | `POST /api/v1/chat` (SSE 스트리밍) | `POST /api/v1/chat/sync` (동기) |
| 인벤토리 | 항상 `[]` (전송 안 함) | 현재 보유 아이템 자동 전송 |
| 세션 | 브라우저 세션마다 `thread_id` | 게임 실행마다 `mc-<uuid>` |

---

## 6. 문제 해결

| 증상 | 원인 / 해결 |
| --- | --- |
| 페이지가 안 열림 | `bash start.sh` 실행 여부·포트(8002) 확인 |
| `오류: ...` 메시지 | 백엔드(8001) 미실행 또는 LLM/벡터DB 키 문제 → 백엔드 로그 확인 |
| 답변이 안 나옴 | `.env`의 `UPSTAGE_API_KEY`·`QDRANT_URL`/`QDRANT_API_KEY` 확인 |
| 참고 위키가 안 붙음 | 일반/범위 밖 질문이라 위키 검색을 거치지 않은 경우(정상) |
| 사이드바 목록 비어 있음 | 첫 대화 전이거나 백엔드 `/api/v1/sessions` 응답 실패 |
