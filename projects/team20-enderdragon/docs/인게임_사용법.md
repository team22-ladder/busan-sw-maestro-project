# 인게임 사용법 — Fabric 코치 모드

> 마인크래프트 안에서 백엔드 코칭 에이전트를 호출하는 **Fabric 모드** 사용법이다.
> 모드 빌드/구조의 개발자 관점은 [../minecraft-mod/README.md](../minecraft-mod/README.md), 기술 전반은 [기술_문서.md](기술_문서.md) 참고.

---

## 1. 사전 준비

- **JDK 21** (`java -version` → 21.x)
- **백엔드 실행 중** — `project_code/`에서 `bash start.sh` → `http://localhost:8001`
  - 백엔드가 꺼져 있으면 코치 호출 시 "백엔드에 연결하지 못했어요" 메시지가 뜬다.

## 2. 실행

### 개발용 (인게임 테스트)
```bash
cd minecraft-mod
./gradlew runClient      # 개발용 마인크래프트 클라이언트 실행 (최초 1회는 다운로드로 수 분)
```
마인크래프트가 뜨면 → **싱글플레이 월드 입장** → 아래 세 가지 진입점으로 코치를 쓸 수 있다.

### 정식 설치 (선택)
```bash
./gradlew build          # 산출물: build/libs/minecraft-coach-<버전>.jar
```
`build/libs/`의 jar와 **Fabric API**를 Fabric Loader가 설치된 마인크래프트 `mods/` 폴더에 넣는다.

## 3. 세 가지 진입점

세 진입점 모두 **동일한 백엔드**를 호출하며, 호출 시 **현재 인벤토리를 자동으로 함께 전송**한다(아래 5장).

### ① 채팅 명령어 `/coach`

채팅창에 직접 질문한다.
```
/coach 이제 뭐 해야 해?
/coach 철 곡괭이 만들고 싶은데 지금 돌 곡괭이밖에 없어
```
- `[코치] 물어보는 중…`(회색) → 잠시 후 `[코치]`(초록) 아래로 답변이 줄 단위로 출력된다.
- 이 메시지는 **나에게만 보이고** 서버(멀티플레이)로 전송되지 않는다.
- `/coach`만 입력하면 사용법 안내가 뜬다.

### ② 코치 GUI 화면 — 기본 키 `K`

게임 채팅과 분리된 별도 창에서 대화한다.
- 입력창에 메시지를 치고 **Enter** 또는 **보내기** 버튼으로 전송.
- 대화 기록이 스크롤로 남고, 창을 닫았다 다시 열어도(K) **같은 세션 기록이 유지**된다.
- `Esc`로 닫는다.

### ③ 할 일 목록 · HUD — 기본 키 `J`

코치 답변의 단계별 TODO가 자동으로 정리된다.
- 코치가 `- ` 불릿으로 안내한 단계를 파싱해 **화면 우측 상단 HUD**에 요약 표시한다(예: `1. 조약돌 수집`).
- **`J`** 를 누르면 **할 일 목록 화면**이 열린다.

| 키 | 동작 |
| --- | --- |
| `↑` / `↓` | 항목 선택 이동 (마지막은 `[ 모두 완료 ]`) |
| `Enter` | 선택 항목 완료 (sparkle 애니메이션 + 사운드) |
| `Esc` | 닫기 |

- 항목을 완료하면 목록·HUD에서 사라진다. 모두 비면 HUD도 사라진다.
- HUD는 인벤토리 등 다른 화면을 열거나 `F1`(HUD 숨김) 상태에서는 표시되지 않는다.

> 키는 마크 **옵션 → 조작 → "마크 코치"** 에서 변경할 수 있다(코치=`K`, 할 일=`J`).

## 4. 사용 시나리오 예시

```
1) /coach 방금 시작했는데 뭐부터 해야 해?
   → 코치가 "지금 할 일" + 단계별 TODO 안내
   → 우측 상단 HUD에 TODO 자동 등록

2) 나무를 캐고 작업대를 만든 뒤
   J 키 → 완료한 항목에서 Enter → 체크 (HUD에서 사라짐)

3) K 키로 GUI를 열어 "철 곡괭이 만들려면?" 이어 질문
   → 같은 세션이라 이전 맥락을 기억해 답변
```

## 5. 인벤토리 자동 인식 (이슈 #18)

코치를 호출할 때마다 모드가 **현재 보유 아이템을 캡처해 백엔드로 함께 보낸다.**
- 캡처 범위: 인벤토리(36) + 방어구(4) + 보조손(1) 중 **비어있지 않은 슬롯**.
- 백엔드는 아이템 ID를 한국어명으로 바꿔 답변에 활용한다 → **"지금 가진 재료로 당장 만들 수 있는 것"**을 우선 안내하고, 부족한 재료의 개수를 짚어준다.
- 그래서 인게임에서는 "돌 곡괭이밖에 없어" 같은 상황 설명을 굳이 적지 않아도 된다.

## 6. 백엔드 주소 설정

기본값은 `http://localhost:8001`. 다른 주소(배포 서버 등)로 바꾸려면 둘 중 하나:

```bash
# 1) JVM 시스템 프로퍼티
./gradlew runClient -Dcoach.backend.url=http://192.168.0.10:8001

# 2) 환경변수
COACH_BACKEND_URL=http://192.168.0.10:8001 ./gradlew runClient
```

- 대화 맥락용 `thread_id`는 게임 실행마다 자동 생성(`mc-<uuid>`)되어, **한 실행 동안** 후속 질문이 이어진다. 게임을 재시작하면 새 세션이 된다.

## 7. 문제 해결

| 증상 | 원인 / 해결 |
| --- | --- |
| `백엔드에 연결하지 못했어요` | 백엔드 미실행 → `project_code/`에서 `bash start.sh`. 주소가 다르면 6장 참고 |
| `백엔드 응답 오류 (HTTP 5xx)` | 백엔드 로그 확인(LLM/벡터DB 키·연결) |
| `답변이 비어 있어요` | 다시 질문. 반복되면 백엔드 LLM 설정 확인 |
| HUD가 안 보임 | TODO가 없거나(`- ` 불릿 답변을 받은 적 없음), `F1`로 HUD를 숨긴 상태 |
| `K`/`J`가 안 먹힘 | 다른 모드와 키 충돌 → 옵션 → 조작에서 재설정 |
