# 백엔드 사용법 — FastAPI 서버 띄우기

웹뷰와 인게임 모드가 공통으로 호출하는 **백엔드(FastAPI)를 클론 → 로컬 실행**하는 절차다. 팀원이 처음 받아서 바로 띄울 수 있도록 정리했다.

> 웹뷰·인게임 모드는 이 백엔드가 **떠 있어야** 동작한다. 클라이언트 사용법은 [웹뷰_사용법.md](웹뷰_사용법.md) · [인게임_사용법.md](인게임_사용법.md), 내부 구조·API는 [기술_문서.md](기술_문서.md) 참고.

> 💡 위키 벡터 **5,923개는 공유 Qdrant Cloud에 이미 적재**돼 있다. 재적재(임베딩)할 필요 없이 접속 정보만 받으면 된다. 각자 로컬엔 **MySQL만** 띄우면 끝.

---

## 사전 준비물

| 항목 | 설명 |
| --- | --- |
| [uv](https://docs.astral.sh/uv/) | 파이썬 패키지·런타임 관리 (Python 3.12 자동 설치) |
| Docker Desktop | 로컬 MySQL 컨테이너용 (실행 중이어야 함) |
| `UPSTAGE_API_KEY` | [Upstage 콘솔](https://console.upstage.ai)에서 발급 — LLM·임베딩용 |
| `QDRANT_URL` · `QDRANT_API_KEY` | **공유 Qdrant Cloud 접속 정보 — 담당자(정선웅)에게 요청** |

---

## 1. 클론

```bash
git clone https://github.com/qlqlrh/asm-team20-ai-study.git
cd asm-team20-ai-study/project_code
```

> #5가 아직 머지 전이라면: `git checkout 'feat/#5'` — 브랜치명에 `#`가 있어 **따옴표 필수**.

## 2. 환경변수 설정

```bash
cp .env.example .env
```

`.env`를 열어 아래 **3개만** 채운다 (나머지는 기본값 그대로 두면 됨):

```dotenv
UPSTAGE_API_KEY=up_xxxxxxxx                          # 필수
QDRANT_URL=https://xxxx.aws.cloud.qdrant.io          # 공유 클라우드 (담당자에게 받기)
QDRANT_API_KEY=xxxxxxxx                              # 공유 클라우드
```

- `DATABASE_URL`은 로컬 docker MySQL 기본값이라 **그대로** 두면 된다.
- `QDRANT_URL`이 채워지면 앱이 자동으로 **클라우드**를 사용한다 (로컬 Qdrant 불필요).

## 3. 로컬 MySQL 기동

```bash
docker compose up -d mysql
```

> Qdrant는 공유 클라우드를 쓰므로 `mysql`만 띄우면 된다. (로컬 Qdrant까지 쓰고 싶으면 `docker compose up -d mysql qdrant`)

## 4. 의존성 설치

```bash
uv sync          # Python 3.12 고정(.python-version) + 패키지 설치
```

## 5. DB 스키마 적용 (Alembic)

```bash
uv run alembic upgrade head     # users / chat_sessions / messages 테이블 생성
```

## 6. 실행

```bash
bash start.sh
#   Backend  → http://localhost:8001   (FastAPI)
#   Frontend → http://localhost:8002   (Streamlit 검증용 웹뷰)
```

## 동작 확인

백엔드만 빠르게 확인 (터미널):

```bash
curl localhost:8001/health                              # → healthy
curl -X POST localhost:8001/api/v1/chat/sync \
  -H 'Content-Type: application/json' \
  -d '{"message":"철 곡괭이 어떻게 만들어?","thread_id":"test-1"}'
# → 위키 근거 기반의 초보 코치 답변(JSON) 이 나오면 정상 ✅
```

웹뷰로 확인하려면 **http://localhost:8002** 접속 후 `"방금 마크 시작했는데 뭐부터 해야 해?"`처럼 물어본다(자세히는 [웹뷰_사용법.md](웹뷰_사용법.md)).

## 트러블슈팅

| 증상 | 원인 / 해결 |
| --- | --- |
| `uv: command not found` | uv 미설치 → `curl -LsSf https://astral.sh/uv/install.sh \| sh` 후 새 터미널 |
| `Cannot connect to the Docker daemon` | Docker Desktop 미실행 → 앱 켜고 재시도 |
| DB 연결 실패 / `Access denied for user 'app'` | MySQL 컨테이너 미기동 → `docker compose up -d mysql`, `docker compose ps`로 `healthy` 확인 |
| `alembic ... Can't connect to MySQL` | MySQL이 아직 준비 전 → 몇 초 뒤 재시도 |
| 챗 답변이 비거나 검색 0건 | `.env`의 `QDRANT_URL`/`QDRANT_API_KEY` 확인 → `curl -H "api-key: $QDRANT_API_KEY" $QDRANT_URL/collections` 로 연결 점검 |
| Qdrant `Unauthorized` | API Key 오타/만료 → 담당자에게 재확인 |
| 포트 충돌(8001·8002·3306) | 기존 프로세스 종료 또는 포트 변경 |
| Streamlit 첫 실행 이메일 프롬프트 | `start.sh`가 headless로 실행하므로 정상적으로 안 뜬다 |
