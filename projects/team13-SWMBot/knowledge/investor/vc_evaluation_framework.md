# VC 투자 심사 5축 프레임워크

## 개요
VC는 스타트업을 팀·시장·제품·트랙션·경쟁 5개 축으로 평가한다. AI 스타트업은 여기에 모델 의존성, API 비용 구조, 규제 리스크라는 고유 변수가 추가된다. 이 프레임워크는 각 축의 판단 기준과 AI 특이 리스크를 통합해 심사 질문을 날카롭게 만든다.

---

## 1축: 팀 (Team)

**핵심 질문:** 이 팀이 이 문제를 풀기에 세상에서 가장 적합한가?

| 평가 항목 | 좋은 신호 | 나쁜 신호 |
|---|---|---|
| Founder-Market Fit | 해당 산업 5년+ 경력 or 직접 고통 경험 | "AI가 유망해서" 진입 |
| 기술 역량 | ML 논문 출판, 오픈소스 기여, 전 직장 AI 프로덕트 | 외주 개발로 MVP 제작 |
| 실행 이력 | 이전 스타트업 경험, B2B 세일즈 성과 | 학교 프로젝트만 존재 |
| 팀 완성도 | CTO+CEO+도메인 전문가 조합 | 기술만 있고 영업/운영 공백 |

---

## 2축: 시장 (Market)

**핵심 질문:** 시장이 충분히 크고 지금 진입할 이유가 있는가?

- **TAM**: 문제를 가진 모든 잠재 고객의 총 지출액
- **SAM**: 현재 제품으로 접근 가능한 시장
- **SOM**: 3-5년 내 현실적으로 점유 가능한 시장

AI 스타트업 시장 과장 패턴:
1. "글로벌 AI 시장 $500B"를 TAM으로 제시 (실제 접근 불가)
2. 경쟁사 존재를 증명하는 수치를 시장 규모로 혼용
3. Bottom-up 근거 없는 Top-down 일변도

---

## 3축: 제품 (Product)

**핵심 질문:** 제품이 실제로 작동하고, 방어 가능한 차별점이 있는가?

AI 제품 특이 평가 기준:
- **모델 성능 vs 기준선**: GPT-4 대비 얼마나 나은가? 측정 가능한 지표가 있는가?
- **파인튜닝/자체 모델 여부**: 범용 API 래퍼인가 vs 도메인 특화 모델인가?
- **데이터 해자**: 독점 데이터 확보 경로가 있는가?
- **UX 차별점**: 동일 모델로도 UX가 핵심 경쟁력인가?

---

## 4축: 트랙션 (Traction)

**핵심 질문:** 고객이 돈을 내고 있는가? 유지되고 있는가?

| 단계 | 요구되는 증거 |
|---|---|
| Pre-seed | 10+ 인터뷰, 3+ LOI/파일럿 |
| Seed | 첫 유료 고객 3-10개, MRR $5K+ |
| Series A | MRR $50K+, MoM 15%+ 성장, NRR 100%+ |

---

## 5축: 경쟁 (Competition)

**핵심 질문:** 기존 해결책 대비 10배 나은가? 방어 가능한가?

- 직접 경쟁사 대비 포지셔닝 명확성
- OpenAI/Google 직접 진입 시 생존 시나리오
- 특허/데이터/네트워크 효과 등 해자 유형 식별

---

## AI 스타트업 고유 리스크

| 리스크 유형 | 설명 | 심사 질문 |
|---|---|---|
| 모델 의존성 | OpenAI API 의존 → 가격 인상/서비스 종료 취약 | 멀티 모델 전략 있는가? |
| API 비용 | 매출 성장과 함께 COGS 폭증 가능 | 마진 유지 시나리오가 있는가? |
| 규제 리스크 | EU AI Act(고위험 AI 2026년 8월 전면 시행), 개인정보보호법 영향 | 컴플라이언스 로드맵 있는가? |
| 모델 범용화 | GPT-5 출시로 차별점 소멸 | 차별점이 모델 외 요소에 있는가? |
| 환각(Hallucination) | 고위험 분야(의료·법률)에서 책임 문제 | 오류 완화 장치가 있는가? |

---

## 체크리스트

- [ ] 팀에 해당 도메인 5년 이상 경험자가 있는가?
- [ ] MVP가 외주 없이 내부 개발되었는가?
- [ ] 시장 규모 산정에 Bottom-up 근거가 있는가?
- [ ] 제품이 단순 API 래퍼를 넘는 차별점을 보유하는가?
- [ ] 유료 고객이 최소 3개 이상 존재하는가?
- [ ] OpenAI 직접 진출 시나리오에 대한 답변이 준비되어 있는가?
- [ ] API 비용 증가에도 흑자 전환 가능한 단가 구조인가?
- [ ] 개인정보·AI 규제에 대한 컴플라이언스 계획이 있는가?
- [ ] 창업팀 지분 구조가 균형 잡혀 있는가? (한 명 80%+ 경계)
- [ ] 경쟁사 대비 10배 개선 지표를 수치로 제시할 수 있는가?

---

## 흔한 레드플래그

- "경쟁사 없음"을 자신 있게 선언하는 팀 (시장 조사 부족의 신호)
- AI 기술 설명에 20분, 고객 문제 설명에 2분 (제품 중심 사고 결핍)
- 트랙션으로 "가입자 수"만 제시하고 유료 전환·유지율 미공개
- 밸류에이션 근거가 "비슷한 회사가 이 가격에 투자받았다"뿐인 경우
- 팀 전원이 같은 학교 동기로만 구성된 경우 (다양성 부족, 외부 검증 부재)
- 제품 데모가 엄선된 케이스만 보여주고 실패 케이스를 숨기는 경우
