# 팀 평가 프레임워크

## 개요
초기 스타트업에서 팀은 제품보다 중요하다. 제품은 피벗할 수 있지만 팀 DNA는 바꾸기 어렵다. AI 스타트업의 경우 기술 역량과 도메인 전문성의 균형, 그리고 "왜 이 팀이 지금 이 문제를 풀어야 하는가"라는 Founder-Market Fit이 핵심 판단 기준이다.

---

## Founder-Market Fit 정의와 판단 기준

**정의**: 창업자가 해당 문제와 시장에서 남다른 인사이트, 네트워크, 경험을 가지고 있어 경쟁자보다 10배 빠르게 실행할 수 있는 상태

### 판단 기준

| 수준 | 조건 | 예시 |
|---|---|---|
| 최상 | 해당 문제를 직접 겪은 당사자 | 10년 의사 출신의 의료 AI 창업 |
| 상 | 해당 산업 5년+ 종사, 의사결정자 레벨 경험 | 제조업 구매팀장 출신의 공급망 AI |
| 중 | 해당 산업 2-3년, 현장 접점 경험 | 3년차 법무팀 출신의 계약서 AI |
| 하 | 업계 경험 없이 리서치만으로 접근 | "법률 AI 시장이 크다"는 분석으로 창업 |
| 위험 | 트렌드만 좇아 진입 | "AI가 핫해서", "ChatGPT 보고 영감 받아서" |

## 이상적인 팀 구성표

| 역할 | 핵심 책임 | 필수 역량 | 없을 때 리스크 |
|---|---|---|---|
| CEO / 비즈니스 | 전략, 세일즈, 투자 유치 | 도메인 경험, 설득력, 의사결정 속도 | 제품이 시장과 단절 |
| CTO / AI Engineer | 제품 개발, 모델 설계 | ML/AI 구현 능력, 시스템 설계 | 외주 의존, 기술 방향성 부재 |
| 도메인 전문가 | 고객 인사이트, 검증 | 해당 산업 현장 경험 | 제품이 현실과 괴리 |
| GTM / 세일즈 | 고객 발굴, 계약 체결 | B2B 영업 경험, 네트워크 | 좋은 제품이 팔리지 않음 |

**시드 단계 최소 구성**: CEO(비즈니스) + CTO(기술) — 나머지는 초기 파트너 또는 고문으로 보완 가능

## AI 스타트업 팀 특이 평가 항목

### ML 역량 수준 구분
AI 스타트업 팀의 기술 깊이는 3단계로 구분된다:

| 수준 | 특징 | 지속 경쟁력 |
|------|------|------------|
| **API 래퍼** | OpenAI/Anthropic API 호출만 | 낮음 — 누구나 복제 가능 |
| **프롬프트 엔지니어링** | 시스템 프롬프트, RAG 파이프라인 | 중간 — 6개월 내 추격 가능 |
| **모델 파인튜닝/연구** | 자체 학습 데이터, 파인튜닝, 논문 | 높음 — 데이터 해자 형성 |

**핵심 질문**: "현재 기술 수준이 어느 단계이며, 왜 그 수준이 이 서비스에 충분한가?"

### AI 모델 안전성 인식
- 환각(Hallucination) 발생 시 처리 방안을 팀이 알고 있는가?
- 사용자 피해 시나리오를 검토했는가? (의료·법률·금융 영역이라면 특히 중요)
- 모델 업데이트 시 서비스 품질 변동 대응 계획이 있는가?

### 체크리스트 (AI 특화)
- [ ] 팀의 AI 기술 수준이 "API 래퍼"를 넘어서는가?
- [ ] ML 엔지니어가 파인튜닝 또는 RAG 구현 경험이 있는가?
- [ ] AI 오류 발생 시 처리 방안이 팀 내에 설계됐는가?
- [ ] AI 제품 방향을 외부 컨설턴트가 아닌 창업자가 직접 결정하는가?

## 위험한 팀 구성 패턴

| 패턴 | 문제점 |
|---|---|
| 기술팀만 있고 영업/비즈니스 부재 | 제품 완성도에만 집중, 고객 발굴 안 됨 |
| 동일 배경 팀 (같은 학교 동기 3인방) | 외부 검증 없는 에코 챔버, 집단사고 위험 |
| 연쇄창업 실패자들만 구성 | 학습 없는 반복 실패 패턴 여부 확인 필요 |
| 시니어 없는 주니어 팀 전원 | 대형 고객 신뢰 획득 어려움, 산업 네트워크 부재 |
| 공동창업자 지분 편중 | 한 명 70%+ 보유 시 나머지 동기 문제 |
| 외주 CTO 또는 파트타임 핵심 인력 | 실행 속도 저하, 제품 소유권 불명확 |

### CTO 풀타임 여부 검증 방법
창업자가 "CTO는 풀타임"이라고 주장할 때 확인할 수 있는 근거:
- GitHub 커밋 이력: 최근 3개월 활동 빈도와 시간대
- LinkedIn 재직 기간: 이전 직장 퇴사 시점과 합류 시점
- 고용 계약서 또는 급여 지급 증빙
- 공동창업자 지분 Vesting 시작일

**질문 예시**: "CTO의 GitHub에서 지난 달 커밋이 몇 개였나요? 이전 직장은 언제 그만두셨나요?"

## 실행력 지표 — 과거 증거 기반 평가

투자자는 "앞으로 잘 할 것"이 아니라 "과거에 잘 한 증거"를 본다.

### 기술 실행력 증거
- GitHub 커밋 히스토리, 오픈소스 기여 내역
- 발표 논문 또는 특허
- 이전 직장에서 프로덕트 출시 이력
- 해커톤 수상, 기술 블로그 팔로워 수

### 비즈니스 실행력 증거
- 이전 스타트업 운영 경험 (성공/실패 무관)
- B2B 계약 성사 이력 (규모 불문)
- 커뮤니티/조직 운영 경험
- 인터뷰에서 "그래서 어떻게 했나요?" 질문 대응 구체성

### 현재 스타트업 실행 속도
- 창업 후 첫 MVP 출시까지 소요 기간
- 첫 고객 확보까지 소요 기간
- 투자 유치 전 무급으로 운영한 기간

## 지분 구조 (Cap Table) 평가

```
[건강한 지분 구조 예시 - 시드 단계]

공동창업자 A (CEO): 40%
공동창업자 B (CTO): 35%
ESOP (직원 스톡옵션 풀): 10%
초기 엔젤/액셀러레이터: 15%

[경계해야 할 구조]
창업자 A: 80%
창업자 B: 10%
기타: 10%
→ B의 동기 문제, 투자 후 희석 갈등 가능성
```

**Vesting 조건 확인**: 4년 클리프 1년 vesting이 표준. 없으면 공동창업자 이탈 리스크.

## 체크리스트

- [ ] 창업팀에 해당 산업 직접 경험자가 있는가?
- [ ] "왜 지금, 왜 이 팀인가"에 설득력 있는 답변이 있는가?
- [ ] CTO가 풀타임이며 내부 인력인가?
- [ ] 기술 역량을 증명하는 과거 증거가 있는가?
- [ ] 비즈니스 실행력(세일즈, 계약)의 과거 증거가 있는가?
- [ ] 지분 구조가 균형 잡혀 있는가? (단독 80%+ 경계)
- [ ] Vesting 조건이 설정되어 있는가?
- [ ] ESOP 풀이 10% 이상 확보되어 있는가?
- [ ] 팀 구성의 공백(영업, 운영 등)을 인지하고 채용 계획이 있는가?
- [ ] 공동창업자 간 역할 구분이 명확하고 갈등 없이 의사결정하는가?

## 흔한 레드플래그

- "저희는 기술팀이라 마케팅은 나중에 채용할 예정입니다" (핵심 기능 공백)
- 창업 이유가 "ChatGPT 써보고 사업 아이디어가 떠올랐다" (Founder-Market Fit 없음)
- CTO가 외주팀이거나 프리랜서인 경우
- 팀 인트로에 고문(Advisor)만 나열하고 실제 풀타임 멤버가 불명확
- 지분 100%가 한 명에게 있는 솔로 창업 (멘토링은 가능하나 팀 위험성 높음)
- 이전 스타트업에 대해 질문하면 "그냥 잘 안 됐어요"라는 성찰 없는 답변
