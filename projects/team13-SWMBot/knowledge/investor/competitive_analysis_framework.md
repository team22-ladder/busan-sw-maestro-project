# 경쟁사 분석 프레임워크

## 개요
경쟁 분석은 단순히 "우리보다 못한 이유"를 나열하는 게 아니라, 시장 구조와 고객 선택 동인을 이해하는 작업이다. AI 스타트업은 빅테크의 직접 진입 가능성과 데이터 해자 구축 여부라는 추가적인 경쟁 변수를 안고 있다.

---

## 경쟁사 3분류

| 구분 | 정의 | AI 스타트업 예시 |
|---|---|---|
| 직접 경쟁 | 동일 문제를 동일 방식으로 해결 | AI 채용 SaaS vs 다른 AI 채용 SaaS |
| 간접 경쟁 | 동일 문제를 다른 방식으로 해결 | AI 채용 SaaS vs 헤드헌팅 서비스 |
| 잠재 경쟁 | 현재는 다른 시장이지만 진입 가능 | LinkedIn, OpenAI 채용 기능 출시 |

**심사 시 주의**: 대부분의 피치덱은 직접 경쟁만 다루고 간접·잠재 경쟁은 회피한다.

---

## 포지셔닝 매트릭스 구성 방법

### 1단계: 고객 구매 기준 2가지 선정
고객 인터뷰 기반으로 실제 구매 결정 요인 2가지를 X축·Y축으로 설정한다.

예시 (AI 고객 서비스 솔루션):
- X축: 응답 정확도 (낮음 → 높음)
- Y축: 구축 복잡도 (높음 → 낮음)

### 2단계: 경쟁사 포지셔닝 표시
```
[포지셔닝 매트릭스 예시]

낮은 구축 복잡도
        |
   [자사] [경쟁사 B]
        |
낮은 <--|--------------> 높은
정확도  |              정확도
        |
   [경쟁사 A]  [기존 방식]
        |
높은 구축 복잡도
```

### 3단계: 화이트 스페이스 식별
자사만이 위치한 사분면이 진정한 차별점 구역이어야 한다.

**경고 신호**: 자사를 모든 축에서 최상위에 배치하는 "우리가 다 이긴다" 매트릭스

## 경쟁사 분석 심층 항목

| 분석 항목 | 확인 내용 |
|---|---|
| 펀딩 현황 | 주요 경쟁사의 총 투자 유치액, 최근 라운드 |
| 고객 수 / ARR | 공개된 고객 수, 인터뷰로 확인된 단가 |
| 제품 성능 | 공개 벤치마크, G2/Capterra 리뷰 |
| 팀 규모 | LinkedIn 인원 수, 기술팀 비율 |
| 이탈 고객 인터뷰 | 경쟁사에서 이탈한 고객이 무엇을 원했는가 |

## AI 스타트업 경쟁 특이점

### 1. OpenAI / Google 직접 진입 리스크

빅테크가 제품과 동일한 기능을 기본 탑재할 가능성을 항상 평가해야 한다.

| 진입 가능성 | 조건 | 대응 전략 |
|---|---|---|
| 높음 | 범용 기능, 데이터 불필요, 대형 시장 | 도메인 특화, 워크플로 통합 |
| 중간 | 특정 산업 지식 필요, 규제 환경 | 규제 대응 전문성, 파트너십 |
| 낮음 | 독점 데이터 필수, 니치 시장 | 데이터 해자 강화 |

**심사 질문 예시**: "ChatGPT가 내년에 동일 기능을 무료로 제공한다면 어떻게 됩니까?"

### 2. 데이터 해자 (Data Moat)

AI 제품의 핵심 방어막은 경쟁사가 복제할 수 없는 데이터 자산이다.

| 해자 유형 | 강도 | 예시 |
|---|---|---|
| 독점 데이터 계약 | 매우 강함 | 특정 병원 EMR 독점 공급 계약 |
| 네트워크 효과 데이터 | 강함 | 유저 증가 → 모델 성능 자동 개선 |
| 레이블링 축적 | 중간 | 도메인 전문가 수년 레이블링 자산 |
| 공개 데이터 파인튜닝 | 약함 | 누구나 접근 가능한 데이터 활용 |

### 3. 모델 성능 경쟁의 빠른 균등화

오늘의 성능 우위는 6-12개월 내 경쟁사가 따라잡을 수 있다. **모델 성능 이외의 방어 장치**를 반드시 확인해야 한다.

- 고객사 내부 시스템과의 깊은 통합 (전환 비용 증가)
- 업무 프로세스에 내재화된 워크플로
- 인증·규정 준수 인프라 (재구축 비용이 높은 영역)

## 흔한 레드플래그

- 경쟁사 비교표에서 자사만 모든 항목에 체크되어 있는 경우
- 간접 경쟁(기존 방식, 스프레드시트 등)을 완전히 무시
- "OpenAI도 경쟁사가 아닙니다"라는 근거 없는 자신감
- 경쟁사 분석이 공개 홈페이지 정보만으로 작성된 경우 (고객 인터뷰 없음)
- 포지셔닝 매트릭스에서 경쟁사들이 모두 한쪽 구석에 몰려 있는 조작된 레이아웃
- 데이터 해자를 주장하지만 구체적인 계약 또는 독점성 근거를 제시 못하는 경우
