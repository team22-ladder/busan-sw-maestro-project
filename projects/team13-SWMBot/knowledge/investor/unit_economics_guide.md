# 유닛 이코노믹스 가이드

## 개요
유닛 이코노믹스는 고객 한 명을 획득하고 유지하는 데 드는 비용과 수익의 비율이다. AI 스타트업은 API 호출 비용이 변동비로 작용해 매출 증가와 함께 COGS가 함께 폭증할 수 있는 구조적 특이점을 가진다. LTV/CAC 비율과 Payback Period를 검증하지 않으면 "성장할수록 손해"인 사업 구조를 놓칠 수 있다.

---

## 핵심 공식

### LTV (고객 생애 가치)
```
LTV = ARPU × Gross Margin % × (1 / Churn Rate)

예시:
- ARPU (월 평균 매출): 50만원
- Gross Margin: 60%
- 월 Churn Rate: 3%
- LTV = 50만원 × 0.60 × (1 / 0.03) = 1,000만원
```

### CAC (고객 획득 비용)
```
CAC = 총 세일즈·마케팅 비용 / 신규 획득 고객 수 (같은 기간)

예시:
- 월 마케팅 비용: 500만원
- 월 세일즈 인건비: 300만원
- 당월 신규 고객: 8개사
- CAC = (500 + 300) / 8 = 100만원
```

### Payback Period (투자 회수 기간)
```
Payback Period = CAC / (ARPU × Gross Margin %)

예시:
- CAC: 100만원
- ARPU: 50만원
- Gross Margin: 60%
- Payback = 100 / (50 × 0.6) = 100 / 30 = 약 3.3개월
```

---

## LTV/CAC 비율 해석표

| LTV/CAC 비율 | 평가 | 투자자 시각 |
|---|---|---|
| 1 미만 | 위험 | 고객 획득할수록 손실 |
| 1 ~ 2 | 주의 | 마진 부족, 사업 개선 필요 |
| 2 ~ 3 | 보통 | 최소 기준, 운영 개선 여지 |
| 3 ~ 5 | 양호 | 투자 고려 수준 |
| 5 이상 | 우수 | 강한 사업 모델, 공격적 투자 가능 |
| 10 이상 | 최상 | 시장 지배적 사업 구조 |

**SaaS 업계 일반 기준**: LTV/CAC ≥ 3, Payback Period ≤ 18개월

## Payback Period 업종별 벤치마크

| 비즈니스 유형 | 권장 Payback Period |
|---|---|
| B2C 소비자 앱 | 3-6개월 |
| SMB 대상 SaaS | 6-12개월 |
| 중견기업 대상 SaaS | 12-18개월 |
| 엔터프라이즈 SaaS | 18-24개월 |
| AI API 플랫폼 | 6-12개월 (데이터 증가에 따른 개선 기대) |

## AI API 비용의 변동비 특이점

일반 SaaS는 인프라 비용이 고정비 성격이 강하지만, AI 스타트업은 **LLM API 호출 비용이 사용량에 정비례**한다.

```
[AI SaaS 원가 구조 예시]

월 매출: 1,000만원
  └ 고정비: 서버 운영 50만원
  └ 변동비: OpenAI API 호출 비용
      - 평균 고객당 월 1만 토큰 × 100개사 = 100만 토큰
      - GPT-4o 기준 $5/1M 토큰 = 약 65만원
  └ 인건비 (COGS 배분): 200만원

Gross Margin = (1,000 - 50 - 65 - 200) / 1,000 = 68.5%
```

고객이 10배 늘면 API 비용도 10배가 된다. 일반 SaaS와 달리 **규모의 경제가 마진 개선으로 이어지지 않을 수 있다.**

## AI 비용 폭증 시나리오

### 시나리오 1: 예상보다 긴 프롬프트 사용
- 고객이 제품을 더 적극적으로 사용할수록 API 호출 증가
- 유저 당 평균 토큰 수가 2배 늘면 마진이 10%p 하락 가능

### 시나리오 2: 무제한 플랜에서 헤비 유저 집중
- "월 정액 무제한" 플랜에서 상위 10% 유저가 비용 70% 발생
- 해결책: 사용량 상한 or Tier별 단가 설계

### 시나리오 3: 가격 경쟁과 API 비용 역전
- 경쟁사 가격 인하 압력으로 구독료 하락 시
- API 비용 > 구독료가 되는 역전 현상 발생 가능

### 대응 지표 확인 항목
```
API 비용 비율 = 월 API 비용 / 월 매출 × 100%

건전한 수준: < 20%
주의 수준: 20-35%
위험 수준: > 35%
```

## 체크리스트

- [ ] LTV, CAC, Payback Period를 각각 계산해서 제시했는가?
- [ ] LTV/CAC 비율이 3 이상인가?
- [ ] Payback Period가 업종 기준 내에 있는가?
- [ ] CAC 계산에 세일즈 인건비가 포함되어 있는가?
- [ ] Gross Margin 계산에 AI API 비용이 포함되어 있는가?
- [ ] 고객 증가에 따른 API 비용 시뮬레이션이 있는가?
- [ ] 헤비 유저로 인한 마진 압박 대응 플랜이 있는가?
- [ ] 월별 Churn Rate가 측정되고 있는가?
- [ ] NRR(Net Revenue Retention)이 100% 이상인가?
- [ ] 현재 마진 구조에서 흑자 전환 시점을 제시할 수 있는가?

## 흔한 레드플래그

- Gross Margin 계산에서 AI API 비용을 누락 (마진 50% → 실제 30%)
- CAC를 광고비만으로 계산하고 세일즈 팀 인건비 제외
- "LTV는 계산 중"이라며 현재 데이터 없이 추정치만 제시
- 월정액 무제한 플랜인데 헤비 유저 비용 분석 없음
- Payback Period 18개월 초과인데 "엔터프라이즈라서 괜찮다"는 근거 없는 주장
- NRR이 80% 미만이면서 성장률로 이를 가리려는 시도
