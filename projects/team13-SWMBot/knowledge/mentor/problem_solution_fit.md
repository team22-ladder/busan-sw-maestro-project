# 문제-해결 적합성 (Problem-Solution Fit)

## 개요
좋은 스타트업은 "우리가 만들고 싶은 것"이 아니라 "고객이 실제로 겪는 고통"에서 출발한다. 문제-해결 적합성은 정의한 문제가 실재하고, 제안한 해결책이 그 문제를 직접적으로 해소하며, 고객이 그 가치에 기꺼이 돈을 낼 의향이 있는지 확인하는 단계다.

---

## 1. 좋은 문제 정의 3조건

| 조건 | 설명 | 나쁜 예 | 좋은 예 |
|---|---|---|---|
| **구체적** | 누가, 언제, 어디서 겪는 문제인지 명확 | "사람들이 학습에 어려움을 겪는다" | "직장인 영어 학습자가 출퇴근 시 30분 이내로 발음 피드백을 받을 방법이 없다" |
| **측정 가능** | 문제의 규모·빈도·비용이 수치로 표현됨 | "많은 사람들이 불편해한다" | "설문 응답자 74%가 주 3회 이상 해당 불편을 경험, 평균 2시간 낭비" |
| **지불 의향** | 고객이 이 문제를 해결하기 위해 돈·시간·노력을 지출할 의사가 있음 | "무료 서비스를 원한다고 했다" | "현재 유사 서비스에 월 2~3만원 지출 중, 더 좋은 대안 있다면 5만원도 가능" |

---

## 2. 문제 → 해결 → 결과 논리 연결

```
[문제 정의]
  └─ 누가 (타겟 고객): 직장인 영어 학습자
  └─ 무엇을 (고통): 실시간 발음 피드백 부재
  └─ 왜 지금 (urgency): 원격근무 증가로 영어 화상회의 급증
         │
         ▼
[해결책]
  └─ 어떻게: AI 음성 분석으로 즉각 발음 교정 제공
  └─ 차별점: 기존 앱 대비 단어→문장 단위 분석 정확도 23% 향상
         │
         ▼
[결과/가치]
  └─ 고객 가치: 매일 10분 연습으로 3개월 내 원어민 발음 유사도 40% 개선
  └─ 비즈니스 가치: 월 구독 전환률 목표 8%
```

논리가 끊기는 구간을 반드시 확인할 것. 기획서에서 화살표가 단순히 그려져 있다고 논리가 연결되는 게 아니다.

---

## 3. 흔한 논리 비약 패턴 3가지

### 패턴 1: 기술 → 가치 비약
> "저희는 GPT-4 기반 AI를 탑재했습니다. 따라서 최고의 학습 경험을 제공합니다."

기술 스택이 곧 고객 가치가 되지 않는다. GPT-4를 쓴다는 것은 수단이지, 고객이 얻는 구체적 결과가 아니다. **"GPT-4를 써서 고객의 어떤 문제가 어떻게 해결되는가?"** 를 명확히 서술해야 한다.

### 패턴 2: 기능 나열 → 해결 비약
> "AI 대화 파트너, 발음 교정, 단어 암기, 진도 관리, 소셜 랭킹 기능을 제공합니다. 따라서 영어 학습 문제를 완전히 해결합니다."

기능이 많다고 문제가 해결되지 않는다. 각 기능이 어떤 세부 문제를 어떤 메커니즘으로 해결하는지 1:1 대응이 필요하다. **"AI 대화 파트너와 발음 교정을 동시에 MVP에 넣으면 무엇을 검증하려는 건가요?"**

### 패턴 3: 수요 가정 비약
> "영어 학습 시장이 5조원이니 0.1%만 점유해도 50억원입니다."

시장 규모는 수요의 존재를 증명하지 않는다. 내 제품에 대한 수요는 실제 고객과의 인터뷰·테스트를 통해서만 검증된다.

---

## 4. 검증 방법

| 검증 방법 | 최소 기준 | 신뢰도 |
|---|---|---|
| 고객 인터뷰 | 타겟 고객 20명 이상 대면/화상 인터뷰 | 중간 (bias 주의) |
| 랜딩 페이지 테스트 | 핵심 가치 제안 1문장 → 이메일 등록 전환률 5% 이상 | 높음 (행동 데이터) |
| 프로토타입 테스트 | Figma/종이 프로토타입으로 5명 태스크 완료율 측정 | 높음 |
| 사전 결제 테스트 | 출시 전 선결제 의향 or 실제 결제 받기 | 매우 높음 |
| 경쟁사 리뷰 분석 | App Store/Reddit 등 기존 제품 불만 패턴 50건 이상 | 중간 |

> **주의:** 지인 인터뷰, "좋을 것 같다"는 답변, 온라인 설문 단독은 검증으로 인정하기 어렵다.

## 체크리스트

- [ ] 타겟 고객이 "누구"인지 직업/상황/빈도로 구체화되어 있는가?
- [ ] 문제의 발생 빈도와 현재 대안 대비 불편함이 수치로 제시되어 있는가?
- [ ] 고객이 현재 이 문제 해결에 돈/시간을 쓰고 있다는 근거가 있는가?
- [ ] 해결책이 문제의 어떤 부분을 어떤 메커니즘으로 해소하는지 서술되어 있는가?
- [ ] 고객 인터뷰 20명 이상을 실제로 진행했는가?
- [ ] 인터뷰 결과에서 "고통"이 반복적으로 나타났는가 (패턴 존재)?
- [ ] 프로토타입 or 랜딩 페이지로 행동 기반 검증을 했는가?
- [ ] 기술 도입이 해결책의 핵심 가치로 직접 연결되어 있는가?

## 흔한 레드플래그

- **문제가 모호하다**: "불편하다", "개선이 필요하다" 수준에서 멈추고 구체적 수치·맥락이 없음
- **고객 인터뷰가 없다**: "시장 조사 자료를 참고했다", "주변 지인이 공감했다" 수준
- **기능이 문제를 해결한다는 논리가 없다**: 기능 목록과 문제 사이에 인과 연결 없이 나열만 존재
- **지불 의향 근거가 없다**: "무료로 쓰다가 유료로 전환할 것"이라는 희망적 가정
- **문제와 해결책이 다른 고객을 대상으로 한다**: 정의한 타겟과 실제 인터뷰한 대상이 다름
- **"AI라서 더 잘 해결된다"는 설명이 없다**: AI를 쓰는 이유가 트렌드 편승인지 기능 필연인지 불분명
