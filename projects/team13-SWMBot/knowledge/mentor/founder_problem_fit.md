# 창업자-문제 적합성 (Founder-Problem Fit)

## 개요
왜 이 팀이 이 문제를 풀어야 하는가? Founder-Problem Fit은 창업자가 해결하려는 문제에 대해 남들보다 유리한 통찰, 접근성, 또는 실행력을 가지고 있는지를 평가한다. AI 트렌드를 보고 뛰어드는 팀과 실제로 그 문제를 겪어본 팀의 제품 깊이는 장기적으로 크게 갈린다.

---

## 1. Founder-Problem Fit 3가지 유형

| 유형 | 설명 | 강점 | 위험 |
|---|---|---|---|
| **직접 경험형** | 창업자 본인이 해당 문제를 직접 겪었거나 겪고 있음 | 고객 고통에 대한 공감과 통찰이 깊음 | 창업자 편향으로 시장 객관화가 어려울 수 있음 |
| **도메인 전문성형** | 해당 산업에서 5년+ 실무 경험, 내부 구조를 잘 앎 | 초기 고객 네트워크, 규제·관행 이해 | 기술 실행력이 부족할 수 있음 |
| **기술 전문성형** | 핵심 기술(ML, NLP 등)에서 세계 수준의 역량 보유 | 남이 만들기 어려운 기술적 해자 구축 가능 | 고객 문제보다 기술 완성도에 집착할 위험 |

> 가장 강한 팀은 두 가지 이상을 조합한다. 예: "의료 현장 10년 경험(도메인) + AI 연구소 출신(기술)"

---

## 2. 창업 동기 강함 vs 약함 신호

| 구분 | 강한 동기 신호 | 약한 동기 신호 |
|---|---|---|
| **동기 출처** | "직접 이 고통을 겪었고 기존 솔루션이 너무 불만족스러웠다" | "AI 시장이 크게 성장 중이라 뛰어들었다" |
| **문제 이해 깊이** | 고객의 워크플로우, 대안, 감정적 고통까지 구체적으로 설명 | 시장 보고서 수준의 일반적 설명 |
| **집착 수준** | "이 문제는 반드시 해결되어야 하고, 내가 아니면 안 된다" | "이 문제도 있고, 안 되면 다른 것도 있다" |
| **고객 접근성** | 타겟 고객 커뮤니티에 이미 속해 있거나 쉽게 접근 가능 | "어떻게 고객을 만날지 모르겠다" |
| **장기 비전** | 10년 후 이 분야를 어떻게 바꿀지 구체적 그림이 있음 | "일단 PMF 찾고 나서 생각하겠다" |

---

## 3. AI 스타트업: 기술 드리븐 vs 문제 드리븐

AI 스타트업에서 특히 중요한 구분이다. 출발점이 다르면 제품의 방향과 성공 확률이 달라진다.

```
기술 드리븐 팀:
  출발점: "우리는 좋은 LLM 모델이 있다 → 어디에 쓸까?"
  특징: 기술 시연이 인상적이나 실제 고객 고통 연결이 약함
  위험: "기술을 위한 기술" → 고객이 원하지 않는 것을 완벽하게 만듦
  신호: 피치에서 기술 설명이 70%, 고객 문제 설명이 30%

문제 드리븐 팀:
  출발점: "이 고통이 너무 크다 → AI가 해결할 수 있을까?"
  특징: 고객 인터뷰 많음, 기술보다 문제 중심으로 설명
  강점: 기술이 진화해도 문제 중심으로 유연하게 피벗 가능
  신호: 피치에서 고객 이야기가 먼저, 기술은 수단으로 설명
```

| 항목 | 기술 드리븐 | 문제 드리븐 |
|---|---|---|
| 제품 설계 기준 | 기술이 할 수 있는 것 | 고객이 원하는 것 |
| 피치 순서 | 기술 → 시장 → 문제 | 문제 → 고객 → 기술 |
| 고객 인터뷰 빈도 | 낮음 | 높음 |
| PMF 도달 속도 | 상대적으로 느림 | 상대적으로 빠름 |
| 장기 방어력 | 기술 해자가 있으면 강함 | 고객 이해 해자 |

## 4. 검증 질문 3가지

멘토가 창업자의 Founder-Problem Fit을 평가할 때 사용하는 핵심 질문들:

**질문 1: "왜 당신이어야 하나요?"**
```
강한 답변 예:
  "저는 10년간 HR 담당자로 일하며 이 비효율을 직접 겪었습니다.
   타겟 고객 200명의 연락처를 이미 가지고 있고,
   전 직장에서 동일한 AI 기술을 다른 방식으로 구현한 경험이 있습니다."

약한 답변 예:
  "AI가 이 분야를 혁신할 것이고 저희 팀이 기술력이 있습니다."
```

**질문 2: "첫 10명의 고객을 어떻게 확보할 건가요?"**
```
강한 답변 예:
  "이미 인터뷰한 20명 중 5명이 '나오면 바로 쓰겠다'고 했고,
   커뮤니티에 테스트 링크를 올려 첫 주에 50명이 가입했습니다."

약한 답변 예:
  "SNS 마케팅과 앱스토어 출시로 자연스럽게 유입될 것 같습니다."
```

**질문 3: "이 문제가 해결되지 않으면 어떻게 되나요?"**
```
강한 답변 예:
  "현재 타겟 기업들은 이 문제로 인해 연간 평균 2,000만원의 비용 손실이 발생합니다.
   저희 고객 인터뷰 결과 이 문제로 3명 중 1명이 이직을 고려한 경험이 있다고 했습니다."

약한 답변 예:
  "불편하기는 하지만 지금도 어떻게든 해결하고 있습니다."
```

## 5. Founder-Problem Fit 평가 매트릭스

| 평가 항목 | 0점 | 1점 | 2점 |
|---|---|---|---|
| 문제 직접 경험 | 없음 | 간접 경험 | 직접 경험 (본인 또는 전 직장) |
| 도메인 지식 | 3개월 미만 | 1~3년 | 3년 이상 |
| 고객 네트워크 | 없음 | 5명 미만 아는 사람 | 20명 이상 즉시 접근 가능 |
| 인터뷰 수행 | 0명 | 1~9명 | 20명 이상 |
| 창업 동기 명확성 | 트렌드 편승 | 일반적 불편함 | 반드시 해결해야 한다는 절박함 |
| 기술 역량 | 외주 의존 | 기본 구현 가능 | 핵심 기술 자체 보유 |

> 총점 8점 이상 = 강한 Founder-Problem Fit
> 총점 4점 이하 = 팀 구성 또는 방향 재검토 필요

## 체크리스트

- [ ] 창업자가 직접 이 문제를 경험했거나, 이 분야에서 3년+ 실무 경험이 있는가?
- [ ] 창업자가 타겟 고객과 직접 소통할 수 있는 네트워크가 있는가?
- [ ] 창업 동기가 "시장 크기" 이전에 "구체적 고통"에서 출발하고 있는가?
- [ ] 창업자가 자신의 서비스를 실제로 사용하는 잠재 고객인가?
- [ ] 고객 인터뷰를 20명 이상 수행했는가?
- [ ] 팀이 기술/도메인/비즈니스 중 최소 2가지 역량을 커버하는가?
- [ ] 경쟁사 대비 "왜 이 팀이 이겨야 하는가"에 대한 명확한 답이 있는가?

## 흔한 레드플래그

- **"AI 트렌드에 편승"**: "GPT 나왔을 때 뭔가 해야겠다 싶었어요"는 동기가 아니다. 기술 트렌드는 시작점이 될 수 없다.
- **창업자가 자신의 서비스 비사용자**: "저는 이 문제를 직접 겪지는 않지만 시장이 크다고 생각했어요" — 가장 위험한 신호.
- **고객 인터뷰를 안 했거나 3명 미만**: 시장 리포트를 고객 이해로 대체하는 팀.
- **"아이디어는 있는데 도메인 전문가를 찾는 중"**: 핵심 도메인 지식 없이 시작한 팀, 조기 피벗 확률 높음.
- **팀 전원이 기술자**: 고객과 소통하고 세일즈할 사람이 없음. 기술은 완벽하나 시장 접점이 없음.
- **공동창업자가 전혀 다른 문제에 집중**: 팀 내부의 Founder-Problem Fit 불일치는 빠른 내부 갈등으로 이어짐.
