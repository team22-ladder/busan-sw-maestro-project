# PMF 검증 방법론 (Product-Market Fit Validation)

## 개요
Product-Market Fit(PMF)은 "제품이 시장의 강한 수요를 충족하고 있는 상태"다. PMF를 달성한 제품은 고객이 자발적으로 계속 사용하고, 이탈을 아쉬워하며, 주변에 추천한다. PMF 이전에 스케일링하는 것은 가장 흔한 스타트업 실패 원인 중 하나다.

---

## 1. PMF 핵심 신호 지표표

| 지표 | PMF 미달 | PMF 진입 신호 | PMF 달성 |
|---|---|---|---|
| **D7 리텐션** | < 10% | 10~25% | > 25% |
| **D30 리텐션** | < 5% | 5~15% | > 15% |
| **월간 Churn Rate** | > 10% | 5~10% | < 5% |
| **NPS** | < 20 | 20~40 | > 40 |
| **Sean Ellis 테스트** | < 20% | 20~40% | > 40% |
| **WoM 비율** | < 10% | 10~30% | > 30% |

> 리텐션 곡선이 "수평 안정화(flatten)"되는 것이 PMF의 가장 강력한 신호다. 계속 하락하는 곡선은 PMF 미달을 의미한다.

## 2. NPS 계산법 및 PMF 수준 기준

**NPS(Net Promoter Score) 계산법:**
```
설문: "이 서비스를 친구/동료에게 추천할 가능성은? (0~10점)"

분류:
  - 추천자 (Promoters): 9~10점
  - 중립자 (Passives): 7~8점
  - 비추천자 (Detractors): 0~6점

계산: NPS = 추천자 비율(%) - 비추천자 비율(%)

예시:
  100명 응답 중 추천자 50명, 중립자 30명, 비추천자 20명
  NPS = 50% - 20% = +30
```

| NPS 범위 | 의미 |
|---|---|
| < 0 | PMF 없음, 제품 근본 재검토 필요 |
| 0 ~ 20 | PMF 미달, 핵심 불만 원인 파악 필요 |
| 20 ~ 40 | PMF 진입 가능성, 지속 모니터링 |
| > 40 | PMF 달성, 스케일링 준비 가능 |
| > 70 | 세계적 수준 (Apple, Netflix 등) |

## 3. Sean Ellis 테스트

**방법:** 실제 사용자에게 다음 질문을 한다.

> "만약 이 제품을 더 이상 사용할 수 없게 된다면 어떤 기분이 들겠습니까?"
> 1. 매우 실망스러울 것이다
> 2. 약간 실망스러울 것이다
> 3. 실망하지 않을 것이다 (딱히 유용하지 않다)
> 4. 이미 사용하지 않는다

**해석:**
```
"매우 실망스럽다" 응답 비율이:
  < 20%: PMF 없음, 핵심 가치 재설계 필요
  20~40%: 잠재적 PMF, 세그먼트 분석 필요
  > 40%: PMF 달성 신호
```

**AI 스타트업 적용 시 주의:** 베타 사용자는 일반적으로 관대하게 응답하는 경향이 있다. 실제 유료 사용자 또는 30일 이상 활성 사용자로 대상을 한정해야 신뢰도가 높다.

## 4. AI 서비스 특이 PMF 지표

일반적인 SaaS 지표 외에 AI 서비스에서 추가로 확인해야 할 지표들:

| AI 특이 지표 | 측정 방법 | 기준치 |
|---|---|---|
| **AI 기능 실사용률** | "AI 기능을 한 번이라도 사용한 사용자" / 전체 가입자 | > 60% |
| **AI 결과 수용률** | AI 제안을 수락/적용한 비율 | > 40% |
| **재사용 주기** | 활성 사용자의 평균 세션 간격 | 서비스 성격에 따라 다름 |
| **AI → 구독 전환률** | 무료 AI 기능 사용자 중 유료 전환 비율 | > 5% |
| **인간 대체 지표** | "AI 없이는 못하겠다"는 응답 비율 (Ellis 변형) | > 40% |
| **피드백 루프 참여율** | AI 결과에 평가(좋아요/싫어요)를 남기는 비율 | > 20% |

## 5. PMF 전·후 행동 차이표

| 구분 | PMF 이전 | PMF 이후 |
|---|---|---|
| **사용 패턴** | 가입 후 며칠 내 이탈 | 주기적 재방문, 습관화 |
| **고객 반응** | "괜찮은 것 같아요" | "이거 없으면 안 돼요" |
| **추천 행동** | 물어봐야 추천 | 자발적 바이럴 공유 |
| **불만 표현** | 조용히 이탈 | 기능 개선 요청, 적극 피드백 |
| **지불 행동** | 무료만 사용 | 유료 전환 자발적 요청 |
| **팀 체감** | 영업/마케팅 강하게 푸시해야 성장 | 사용자 스스로 홍보 |

## 6. PMF 측정 타이밍

```
측정 시점 권장안:

Week 1: 가입 즉시 → 첫인상 NPS
Week 4: 4주 사용 후 → Sean Ellis 테스트
Week 8: 8주 사용 후 → D7/D30 리텐션 코호트 분석
Week 12: 3개월 사용자 → 심층 인터뷰 + 이탈 원인 분석

주의: 베타 출시 직후 1~2주 데이터로 PMF를 판단하지 말 것.
초기 얼리어답터의 열정 편향이 존재한다.
```

## 흔한 레드플래그

- **가입자 수 = PMF 오해**: 가입자 1만 명은 인지도 지표다. PMF 지표가 아니다. D30 리텐션 5% 미만이라면 9천500명이 의미없다.
- **D30 리텐션 5% 미만에서 스케일링 시도**: 새는 바구니에 물을 붓는 것이다. CAC를 아무리 낮춰도 수익을 낼 수 없다.
- **PR/미디어 트래픽을 PMF로 착각**: 출시 직후 미디어 버즈로 유입된 트래픽은 리텐션 지표를 왜곡한다. 최소 4~6주 후 코호트 데이터를 봐야 한다.
- **베타 사용자만으로 PMF 판단**: 베타 사용자는 얼리어답터 편향이 강하다. 일반 사용자 100명 이상 데이터가 필요하다.
- **정성 피드백만으로 PMF 선언**: "다들 좋다고 했어요"는 데이터가 아니다. 수치로 검증해야 한다.
- **MoM 성장률을 PMF 근거로 사용**: 적극적 마케팅 투자로 만들어진 성장은 PMF가 아니다. 오가닉/WoM 성장 비율을 분리해야 한다.
