# MVP 기술 범위 설정 및 Build vs Buy 판단

## 개요
AI 스타트업 MVP에서 가장 흔한 실수는 직접 구현할 것과 사야 할 것을 구분하지 못하는 것이다. 핵심 경쟁력이 아닌 영역에 개발 리소스를 낭비하면 6개월 안에 시장 검증 자체가 불가능해진다. CTO는 팀 규모와 기간을 고려해 현실적인 구현 범위를 설정해야 한다.

---

## Build vs Buy 판단 기준

| 판단 기준 | Build (직접 구현) | Buy (솔루션 구매/사용) |
|-----------|------------------|----------------------|
| 핵심 경쟁력 여부 | 우리만의 차별점이 여기서 나온다 | 일반적 기능, 경쟁사도 동일하게 씀 |
| 구현 기간 | 팀이 2주 이내 완성 가능 | 2주 이상 필요한 복잡도 |
| 장기 비용 | 운영 비용 < 솔루션 구독료 | 운영 비용 > 솔루션 구독료 |
| 데이터 민감도 | 외부에 보내면 안 되는 데이터 | 민감하지 않거나 암호화 가능 |
| 커스텀 필요도 | 우리 요구사항에 맞는 솔루션 없음 | 기성 솔루션이 90% 이상 충족 |
| 유지보수 역량 | 내부 전문 인력 보유 | 해당 기술 전문가 없음 |

### MVP Buy 권장 목록 (직접 구현 하지 말 것)

| 영역 | 권장 솔루션 | 이유 |
|------|-----------|------|
| LLM API | OpenAI API, Anthropic API | 자체 LLM 훈련은 수억 원 + 수개월 |
| STT (음성 인식) | OpenAI Whisper API, Google Speech-to-Text | 자체 구현 시 수천 시간 데이터 필요 |
| TTS (음성 합성) | ElevenLabs, OpenAI TTS, Google TTS | 자연스러운 음성 생성은 고난이도 |
| 인증/Auth | Firebase Auth, Clerk, Auth0 | 보안 취약점 리스크, 구현 1~2주 vs 1일 |
| 결제 | Stripe, 토스페이먼츠 | PCI DSS 준수, 직접 구현 불가에 가까움 |
| 이메일 | SendGrid, AWS SES | SMTP 직접 운영 = 스팸 필터 지옥 |
| 벡터 DB | Pinecone, Supabase pgvector | 직접 구현 의미 없음 |
| 모니터링 | Datadog, Sentry, LangSmith | 내부 구축은 수개월 소요 |

## 팀 규모별 6개월 현실적 구현 범위

| 팀 규모 | 기술 구성 | 6개월 현실적 범위 |
|---------|---------|----------------|
| 1~2인 | 풀스택 1명 or AI+풀스택 | MVP 웹앱 1개, API 연동, 기본 인증, 핵심 AI 기능 1~2개 |
| 3~4인 | 백엔드 1, 프론트 1, AI 1, (기획 1) | 웹앱+모바일 중 하나, AI 파이프라인, 관리자 페이지 기초 |
| 5~8인 | 백엔드 2, 프론트 2, AI 1, DevOps 1, PM 1 | 웹+앱, AI 기능 3~4개, 대시보드, 결제 연동 |
| 9인 이상 | 각 파트 전문화 | 본격적인 제품 개발 가능, AI 파인튜닝 시도 가능 |

**주의**: 위 표는 집중 개발 기준이며, 병가·이직·기획 변경 시 30~50% 버퍼 필요

## AI 기능 구현 난이도 및 기간

| AI 기능 | 난이도 | 예상 기간 (1~2인 AI 엔지니어) | 전제 조건 |
|---------|--------|---------------------------|----------|
| 기본 챗봇 (API 연동) | 하 | 1~2주 | LLM API 접근권 |
| RAG 기반 문서 Q&A | 중 | 2~4주 | 문서 데이터, 벡터 DB |
| 음성 입출력 파이프라인 | 중 | 3~5주 | STT/TTS API, 스트리밍 처리 |
| 코드 생성/리뷰 도우미 | 중 | 3~6주 | 코드 평가 기준 설계 |
| 이미지 분석/생성 기능 | 중 | 2~4주 | Vision API, 프롬프트 설계 |
| 추천 시스템 (협업 필터링) | 중~상 | 1~3개월 | 사용자 데이터 최소 1만 건 |
| 감정 분석/분류 모델 | 중~상 | 1~2개월 | 학습 데이터 수천 건 필요 |
| 파인튜닝 모델 (LoRA 등) | 상 | 2~4개월 | GPU 서버, 수천~수만 건 데이터 |
| 자체 언어 모델 훈련 | 매우 상 | 6개월+ | 수억 원 비용, 대규모 팀 |
| 실시간 개인화 추천 | 상 | 3~6개월 | 실시간 데이터 파이프라인, ML 인프라 |

## 과대평가되는 AI 기능 패턴

```
흔한 MVP 기획서의 과도한 요구사항 예시:

기획서: "사용자의 발음을 실시간으로 분석하여 즉각 교정해 줍니다"
현실:
  - Whisper STT 지연: 평균 1~3초 (짧은 문장 기준)
  - 발음 분석 모델: 별도 음소(phoneme) 분류 모델 필요
  - 실시간 교정 UX: 말하는 도중 교정 = 추가 구조 설계 필요
  - 실현 가능 MVP: 문장 완성 후 2~4초 내 피드백 (실시간 X)
```

### 동시에 달성 불가능한 트릴레마

```
          실시간(저지연)
               △
              / \
             /   \
            /     \
    고정밀도 --------- 저비용

셋 중 두 가지만 가능:
- 실시간 + 고정밀도 = 고비용 (GPT-4 + 병렬 처리)
- 실시간 + 저비용  = 정밀도 타협 (소형 모델)
- 고정밀도 + 저비용 = 배치 처리 (비실시간)
```

## 흔한 레드플래그

- **"실시간 + 고정밀 + 저비용" 동시 요구**: 트릴레마 인식 없이 세 가지를 모두 요구하는 기획
- **MVP에 파인튜닝 포함**: 파인튜닝은 기본 서비스 검증 이후에 해야 하는 최적화 단계
- **2인 팀이 앱+AI+어드민+대시보드 6개월 계획**: 현실적으로 둘 중 하나도 완성하기 어려움
- **"AI가 알아서 학습합니다"**: 강화학습·온라인 학습은 고난이도. MVP 범위 외
- **직접 구현 이유가 없는 빌드 결정**: "인증은 보안상 직접 만들어야 해요" = Auth0/Firebase를 모르거나 직접 구현 리스크를 과소평가
- **경쟁사 기능 모두 포함**: 경쟁사가 5년간 만든 기능을 6개월에 구현하겠다는 계획
