# AI 서비스 비용 구조 분석

## 개요
AI 스타트업의 가장 큰 실패 원인 중 하나는 비용 과소 추정이다. LLM API, STT, TTS 비용은 사용자 수가 늘수록 선형 이상으로 증가하며, 초기에 비용 구조를 설계하지 않으면 스케일 시점에 유닛 이코노믹스(Unit Economics)가 역전된다.

## 실제 비용 시뮬레이션

### 예시: 영어 스피킹 학습 앱 (MAU 1만 명)

```
가정:
- 사용자 1인당 일 평균 사용 시간: 10분
- 1분당 발화: 약 100 단어 (영어 기준)
- STT → LLM 피드백 → TTS 파이프라인
- 월 활성 사용자(MAU): 10,000명
- 일 활성 사용자(DAU): 2,000명 (MAU의 20%)

[STT 비용]
  DAU 2,000명 × 10분 × $0.006/분 = $120/일
  월 비용: $120 × 30일 = $3,600/월

[LLM 피드백 비용 (GPT-4o-mini)]
  입력: 사용자 발화 100 토큰 + 시스템 프롬프트 200 토큰 = 300 토큰/요청
  출력: 피드백 150 토큰/요청
  요청 수: DAU 2,000명 × 20회/일 = 40,000회/일
  입력 비용: 40,000 × 300 / 1,000,000 × $0.15 = $1.8/일
  출력 비용: 40,000 × 150 / 1,000,000 × $0.60 = $3.6/일
  LLM 월 비용: ($1.8 + $3.6) × 30 = $162/월

[TTS 비용 (피드백 읽어주기)]
  피드백 평균 100글자 × 40,000회/일 = 4,000,000 글자/일
  TTS-1 비용: 4,000,000 / 1,000,000 × $15 = $60/일
  월 비용: $60 × 30 = $1,800/월

[총 AI 비용]
  STT: $3,600 + LLM: $162 + TTS: $1,800 = 월 $5,562 (약 750만 원)
  사용자 1인당 비용: $5,562 / 10,000 = $0.56/월 (약 750원)

→ 월 구독료 $9.99라면 수익성 있음
→ 월 구독료 $0.99라면 AI 비용만으로 적자
```

### Claude Sonnet 4.6 vs Claude Haiku 4.5 비용 비교

```
동일 조건에서 Claude Sonnet 4.6 사용 시:
  입력: 40,000 × 300 / 1,000,000 × $3.00 = $36/일
  출력: 40,000 × 150 / 1,000,000 × $15.00 = $90/일
  LLM 월 비용: ($36 + $90) × 30 = $3,780/월

Claude Haiku 4.5 사용 시:
  입력: 40,000 × 300 / 1,000,000 × $1.00 = $12/일
  출력: 40,000 × 150 / 1,000,000 × $5.00 = $30/일
  LLM 월 비용: ($12 + $30) × 30 = $1,260/월

Haiku 4.5 대비 Sonnet 4.6: 약 3배 비용 차이
→ 단순 분류·요약 작업은 Haiku 4.5로, 복잡한 추론은 Sonnet 4.6으로 계층화 권장
```

## 비용 최적화 전략

| 전략 | 절감 효과 | 구현 복잡도 | 주의사항 |
|------|---------|-----------|--------|
| 모델 계층화 (Model Tiering) | 60~90% 절감 | 중간 | 품질 하락 구간 모니터링 필요 |
| 의미론적 캐싱 (Semantic Cache) | 20~40% 절감 | 높음 | 캐시 유효기간, 유사도 임계값 설정 |
| 프롬프트 압축 | 10~30% 절감 | 낮음 | 너무 줄이면 품질 저하 |
| 배치 처리 (Batch API) | 50% 절감 | 중간 | 실시간 응답 불가한 작업만 적용 |
| 컨텍스트 윈도우 관리 | 10~40% 절감 | 중간 | 긴 대화 히스토리 요약 처리 |

### 모델 계층화 예시

```python
def select_model(task_type: str, complexity_score: float) -> str:
    """
    task_type: 'greeting', 'qa', 'analysis', 'code_review'
    complexity_score: 0.0~1.0 (자동 분류 또는 규칙 기반)
    """
    if task_type == 'greeting' or complexity_score < 0.3:
        return "gpt-4o-mini"        # 단순 작업: $0.15/1M 토큰
    elif complexity_score < 0.7:
        return "gpt-4o"             # 중간 작업: $2.50/1M 토큰
    else:
        return "gpt-4"              # 복잡 작업: $10/1M 토큰
    # 결과: 평균 비용 70~80% 절감 가능
```

## 흔한 레드플래그

- **AI 비용 계산 없음**: "LLM API는 저렴하다"라는 막연한 인식. 스케일 시 수천만 원/월 가능
- **모든 기능에 Claude Opus 4.8**: 간단한 분류·요약에 최고가 모델 사용 = 불필요한 비용 낭비
- **무제한 무료 체험**: Rate Limit 없는 무료 플랜은 런칭 첫날 예산 초과 위험
- **한국어 토큰 비용 과소평가**: 한국어는 영어 대비 토큰 수 1.5~2배 더 소모
- **TTS 비용 누락**: STT와 LLM 비용만 계산하고 TTS를 빠뜨리는 경우
- **배치 처리 미검토**: 실시간 불필요한 작업(보고서 생성 등)에 실시간 API 사용
