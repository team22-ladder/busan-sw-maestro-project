## 붙여넣기용 원문

발표 전 개인 메모

오늘 서버를 미니PC에서 열어 보니 노트북 브라우저에서 127.0.0.1로 접속하면 안 된다는 점을 다시 확인했다. 발표 때는 반드시 Tailscale IP로 접속해야 하고, 백엔드는 0.0.0.0으로 열어야 한다. 6월 10일 밤에 한 번 더 실행 명령을 점검하자.

내가 해야 할 일은 세 가지다. 첫째, PR 설명에 Solar API를 쓰는 구간과 쓰지 않는 구간을 분리해서 적는다. 6-1 분석은 Solar를 쓰고, 6-2 라우팅과 승인 저장은 규칙 기반이라고 설명해야 한다. 둘째, 발표 전에 UPSTAGE_API_KEY가 실제로 들어가 있는지 확인한다. 셋째, 저장소 탭에서 tasks, calendar_events, risk_logs가 실제로 채워지는지 확인한다.

불안한 부분도 있다. Solar 응답이 JSON 형식으로 깨지면 분석 결과가 이상해질 수 있다. 네트워크가 느리면 분석 버튼을 누른 뒤 대기 시간이 길어질 수 있다. 발표 중에 저장소 DB가 이전 테스트 데이터로 지저분하면 결과 설명이 헷갈릴 수 있으니, 시작 전에 DB 상태를 정리하자.

기억할 것: 6-3 선호 확인 화면은 아직 목데이터라서 실제 Solar API 결과라고 말하면 안 된다.
